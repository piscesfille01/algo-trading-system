# 백테스트 사용 가이드

## 🎯 워크플로우

### 1단계: 과거 날짜 추천 확인
```bash
# 과거 날짜로 pick.js 실행
node scripts/pick.js 2026-03-19

# 결과를 백테스트용으로 저장
node scripts/pick.js 2026-03-19 | node scripts/backtest-save-picks.js 2026-03-19
```

### 2단계: 진입/패스 결정 입력

#### 진입한 경우
```bash
# 목표가 도달
node scripts/backtest-add.js 2026-03-19 NVDA entered 118.50 124.00 115.00 hit 123.80

# 손절
node scripts/backtest-add.js 2026-03-19 AMD entered 120.00 130.00 115.00 stop 114.50

# 아직 보유 중 (현재가 입력)
node scripts/backtest-add.js 2026-03-19 TSLA entered 180.00 190.00 175.00 holding 185.00
```

#### 진입 안 한 경우
```bash
node scripts/backtest-add.js 2026-03-19 INTC passed "score too low"
node scripts/backtest-add.js 2026-03-19 META passed "market conditions bad"
node scripts/backtest-add.js 2026-03-19 AAPL passed "already holding too many"
```

### 3단계: 분석 실행
```bash
# 5건 이상 쌓이면 분석 실행
node scripts/backtest-analyze.js
```

## 🔄 자동 분석 트리거

### 케이스 수에 따라 자동으로 분석:
- **5-9건**: 기본 통계만 (점수별 승률)
- **10-19건**: 패턴 분석 추가
- **20건+**: 고급 분석 (섹터별, 시장 환경별 등)

### 개선점 발견 시:
1. 나: insights.json에 기록
2. 나: 사용자에게 알림
3. 사용자: 승인/거부
4. 나: 승인 시 pick.js 코드 자동 수정

## 💡 팁

### 효율적인 백테스트 방법

1. **연속된 날짜로 테스트**
   ```bash
   # 3월 한 달 전체
   for date in 2026-03-{01..31}; do
     node scripts/pick.js $date | node scripts/backtest-save-picks.js $date
   done
   ```

2. **주요 시장 환경별로 분류**
   - 상승장: 2025-11-01 ~ 2025-12-31
   - 하락장: 2026-01-01 ~ 2026-03-31
   - 혼조장: 2025-09-01 ~ 2025-10-31

3. **최소 30건 이상 추천**
   - 통계적 의미 있는 샘플
   - 각 점수 구간별 3건 이상

4. **다양한 섹터 포함**
   - 기술주만 테스트 X
   - 금융, 에너지, 헬스케어 등 골고루
